Getting Started

Overview

What this guide covers, what you need before starting, how long it takes, and how the final system is structured.

Estimated total time: 45–60 minutes Beginner to Intermediate

What This Guide Covers

This documentation provides a complete, step-by-step guide to deploying Odoo on Oracle Cloud Infrastructure (OCI) using a free-tier or paid VM running Ubuntu 24.04 LTS.

Starting immediately after VM creation and SSH access, it covers:

ChapterTopicTime
1Server preparation, system packages, PostgreSQL, directory layout10–15 min
2Odoo Community & Enterprise installation, Python virtual environment15–20 min
3Odoo configuration file (odoo.conf), memory tuning10 min
4systemd service creation, auto-start, journald logging5–10 min
5Nginx reverse proxy, WebSocket, static caching10–15 min
6Domain DNS configuration and propagation verification5–30 min
7HTTPS via Let's Encrypt / Certbot, automatic renewal10 min
8Oracle Security List + iptables firewall configuration10–15 min
—Helper commands, troubleshooting reference, appendix5 min
What is NOT covered

This guide does not cover Oracle VM creation, SSH key setup, initial Ubuntu access, or Odoo Enterprise licence purchase. It begins at the point where you already have SSH access to a running Ubuntu 24.04 VM.

Prerequisites

☁️
Oracle Cloud Account
A free or paid OCI account with a VM instance running Ubuntu 24.04 LTS
🔑
SSH Access
A working SSH connection to the VM with a user that has sudo privileges
🔗
Domain Name
A domain name you control, registered at any registrar (required for HTTPS)
📦
Odoo Enterprise Access
An active Odoo Enterprise subscription with a GitHub account linked for Enterprise repository access
🌐
Public IP
The public IP address of your Oracle VM (visible in the OCI console)
🐧
Linux Familiarity
Basic comfort with the Linux command line, file editing, and SSH
Community-only deployment

If you only need Odoo Community (open source), you can skip the Enterprise repository step. Simply omit the Enterprise path from addons_path in odoo.conf.

Final Architecture

At the end of this guide, your system will look like this.

Complete stack: Internet traffic enters through Nginx, which proxies to Odoo running on localhost, connected to PostgreSQL via local socket.

Note

A best practice, shortcut, or optional improvement that makes the deployment better.

Warning

A potential problem or action that could break your deployment if done incorrectly.

Danger
